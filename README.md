# OpenEMR Hospital Scheduling System -- End-to-End BA/PM Portfolio Project

## 1. Project Overview

The OpenEMR Scheduling System project modernizes a hospital's
appointment workflow by allowing receptionists, nurses, physicians, and
labs to manage patient data in a single electronic health record (EHR)
system.

## 2. Business Problem

The hospital used manual scheduling, which caused: - Double bookings\
- Lost patient records\
- Delayed lab results\
- Poor communication between departments

## 3. Project Methodology

**Hybrid Approach: Waterfall + Agile** - Planning & requirements:
Waterfall\
- Build & testing: Agile Scrum (2-week sprints)

## 5. Stakeholders

-   Receptionist\

-   Physicians\
-   Lab Technicians\
-   IT/Developers\
-   Product Owner\
-   Scrum Master

## 6. High-Level Workflow

1.  Patient check-in\
2.  Vitals collected\
3.  Physician consultation\
4.  Lab order\
5.  Results uploaded to patient portal

## 7. Sample User Story

**As a receptionist, I want to view the scheduling calendar so that I
can create or edit patient appointments.**

### Acceptance Criteria

-   Valid login\
-   Access to calendar tab\
-   Able to select facility\
-   Successful appointment creation
